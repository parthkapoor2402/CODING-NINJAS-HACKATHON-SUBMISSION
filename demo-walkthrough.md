# Demo Walkthrough — Mobile-Ready CivicResolve

**Duration:** ~12 minutes · **Account:** `demo-citizen@local.dev` unless noted

---

## 1. First launch (2 min)

1. Open app → splash → onboarding persona (**Commuter**).
2. Acknowledge location, camera, notification education (no real permissions required in demo).
3. **Sign in** as citizen → land on **Home** (`home-hero`, Around you map/feed).

**Talk track:** Transparency — neighbors see how evidence and location help crews.

---

## 2. Report an issue (3 min)

1. Tap **primary CTA** → evidence step.
2. **Upload photo** from gallery (or use text-only fallback if camera blocked).
3. Enter description (10+ chars) → watch **AI suggestions** or amber unavailable fallback.
4. Set category/severity → **location** (allow GPS or use manual pin if denied).
5. Review — note **duplicate warning** if near existing pothole; link to **support existing**.
6. Submit → success screen.

**Talk track:** Efficiency — duplicate redirect avoids double tickets; AI assists but never blocks.

---

## 3. Community & trust (2 min)

1. **Nearby** → sort by urgency → open issue detail.
2. **Support existing** on a neighbor’s report (parent account for full corroboration credit).
3. **Community** tab → verify a pending report (`report-003`).
4. **Track** → see timeline progression on your reports.

**Talk track:** Accountability — verification before rewards and routing.

---

## 4. Rewards & family (2 min)

1. **Rewards** → leaderboard, verified points, catalog lock/unlock.
2. Sign in **Youth demo** → **Family** tab → supervised restrictions, safe contribution list.

**Talk track:** Youth safety — no partner redemption without parent.

---

## 5. Admin ops (3 min) — `demo-admin@local.dev`

1. **Dashboard** — open queue, SLA alerts, duplicate signals.
2. **Queue** → filter → open **report-001** → assign crew.
3. **Moderation** — suspicious vs abuse queues; merge duplicate tool.
4. **Analytics** — response times, category trends, abuse flags, predictions.
5. **Hotspots** — ward cards with trend chips.

**Talk track:** Operational usefulness — every metric maps to transparency, accountability, or efficiency.

---

Refresh browser or restart dev server to reset in-memory mock state.
